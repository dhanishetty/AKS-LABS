### Setup Azure CLI
```bash
az login --use-device-code
```

**To register preview features.**


```bash
az extension add --name aks-preview
```

`aks-preview` is the extension that:
- Extends the az aks command group with:
- New subcommands
- New flags/parameters
- Support for new AKS features before GA

you get:
- Extra commands under `az aks`, like:
- `az aks mesh`, `z aks approuting`, or new `az aks nodepool` options (depending on version).
- Extra parameters on existing commands, e.g.:
- New flags on `az aks create`, `az aks update`, `az aks nodepool add`, etc.
- Access to preview-only features that are not yet in the stable CLI.
The docs often say things like:
“This parameter is only available with the `aks-preview` extension.”

### Setup Resource Group

```bash
RAND=$RANDOM
export RAND
echo "Random resource identifier will be: ${RAND}"
```

```bash
export LOCATION=eastus

export RG_NAME=myresourcegroup$RAND
```

```BASH
az group create \
--name ${RG_NAME} \
--location ${LOCATION}
```

### Setup Resources

**To save your user object ID to a variable.**

```bash
export USER_ID=$(az ad signed-in-user show --query id -o tsv)
```

**To deploy Bicep template into the resource group.**

```bash
az deployment group create \
--resource-group ${RG_NAME} \
--name ${RG_NAME}-deployment \
--template-uri https://raw.githubusercontent.com/azure-samples/aks-labs/refs/heads/main/docs/getting-started/assets/aks-labs-deploy.json \
--template-uri https://raw.githubusercontent.com/Azure-Samples/aks-labs/refs/heads/main/docs/getting-started/assets/k8s-aks-fundamentals/main.json
--parameters userObjectId=${USER_ID} \
--no-wait
```

### Create an Azure OpenAI account and `gpt-4.1-mini` model

```bash
# Set the Azure OpenAI account name
export AOAI_NAME=myopenai$RAND

# Create the Azure OpenAI account
AOAI_ID=$(az cognitiveservices account create \
--resource-group $RG_NAME \
--location $LOCATION \
--name $AOAI_NAME \
--custom-domain $AOAI_NAME \
--kind OpenAI \
--sku S0 \
--assign-identity \
--query id -o tsv)

# --name $AOAI_NAME -> myopenai12345
# --custom-domain $AOAI_NAME -> https://myopenai12345.openai.azure.com 
# --sku S0 -> S0 is the standard production tier. 
# --query id -o tsv) Extracts only the resource ID and stores it in:


# Create a deployment for the gpt-4.1-mini model
az cognitiveservices account deployment create \
-n $AOAI_NAME \
-g $RG_NAME \
--deployment-name gpt-4.1-mini \
--model-name gpt-4.1-mini \
--model-version 2025-04-14 \
--model-format OpenAI \
--sku-capacity 200 \
--sku-name GlobalStandard

# --sku-capacity 200 -> tokens per minute your deployment can process, how many requests per second it can handle, how much compute
# --sku-name GlobalStandard -> type of capacity
```
